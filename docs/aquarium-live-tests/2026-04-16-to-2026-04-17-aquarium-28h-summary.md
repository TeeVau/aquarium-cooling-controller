## Fault / Plausibility Observation

The export contains `66` fan-fault intervals totaling about `2240 s`
(`37.3 min`).

| Phase | Fault intervals | Total duration |
|---|---:|---:|
| Light on | `2` | `20 s` |
| Light off | `64` | `2220 s` |

The main fault cluster occurred during the morning low-PWM region around
`07:00` to `08:34` on `2026-04-17`.

Typical fault context:

| Signal | Typical range |
|---|---:|
| Fan PWM | `12% .. 14%` |
| Expected RPM | about `187 .. 223` |
| RPM tolerance | about `+/-22 .. +/-27` |
| Measured RPM | often `180`, `210`, or `240` |

Interpretation: the current plausibility check is too sensitive near the
minimum fan operating region. These events look like low-PWM tach quantization
or tolerance noise rather than real cooling failures.

Follow-up is tracked in GitHub issue `#1`.

## Air-Assist Observation

The second light-on phase shows that air-assist responds to warm lid air and
keeps airflow active. However, because water temperature continued to fall from
about `24.25 C` to `23.19 C`, future tuning should consider a water guard for
air-assist.

Potential future policy options:

- reduce air-assist when water is below target
- disable air-assist below `target - 0.2 C`
- cap air-assist PWM when water is already below target

This should not be changed immediately while OTA is unavailable. Continue
collecting installed logs first.

Follow-up is tracked in GitHub issue `#2`.
